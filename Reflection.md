In this ACC102 Track 2 mini assignment, I completed a Python-based financial data analysis project using real stock data and published the whole project on GitHub. This reflection covers all required aspects, including the analytical purpose, data description, methods, findings, limitations, learning experience, and AI disclosure.
The analytical problem of this project is to analyse the long-term stock performance of Microsoft, including monthly return trends and trading volume changes. The intended audience includes course instructors, classmates, and learners who study Python financial data analysis. This project demonstrates a standard reproducible workflow of academic financial data analysis, which is consistent with the practical requirements of ACC102 and helps to show the application of theoretical knowledge in real financial scenarios.
The dataset used in this project is monthly stock data of Microsoft obtained from the WRDS CRSP monthly stock file. The key variables include trading date, monthly total return, and trading volume. This dataset is selected because Microsoft is a representative large-cap technology listed company with complete, long-term historical data from its IPO in April 1986 to December 2024. The stable continuous data perfectly meets the project requirements of time-series analysis practice in the course, and its typicality in the technology industry also makes the analysis results more representative and referenceable.
In this project, I applied multiple Python methods taught in ACC102 lectures and tutorials. I used the wrds library to establish a secure connection with the WRDS database and extract target data through standard SQL query statements, ensuring the accuracy and rationality of data acquisition. I adopted pandas for data preprocessing, including datetime format conversion to ensure the consistency of time variables and missing value elimination to avoid the impact of invalid data on analysis results. I used matplotlib to build a dual-axis time-series visualisation for return and volume trends, making the data changes more intuitive. I also calculated core financial metrics consisting of cumulative return, annualised return, return volatility and maximum drawdown, which comprehensively reflect the risk and return level of Microsoft stock.
The main outputs of this project include a fully runnable sequential Python notebook, a standard dual-axis trend chart that clearly shows the changes of monthly return and trading volume, a statistical summary table that reflects the basic characteristics of the dataset, and a set of calculated financial risk-return indicators. The main insight is that Microsoft has maintained an overall positive long-term return trend since listing, and its trading volume fluctuations are correlated with the volatility of stock returns in historical periods—usually, higher return volatility is accompanied by an increase in trading volume, which is consistent with the basic laws of the stock market.
This project has clear limitations and reliability issues. The analysis only covers return and volume variables without involving corporate fundamental indicators (such as revenue, profit margin) or macroeconomic factors (such as interest rate changes, economic cycles), which may lead to incomplete analysis of stock performance drivers. The research does not exclude the impact of extreme market shocks (such as financial crises), which may affect the stability of the analysis results. In addition, no peer benchmark comparison is added, so it is impossible to evaluate Microsoft's stock performance relative to other technology companies. Possible future improvements include supplementing fundamental financial ratio analysis to explore the internal drivers of stock performance, adding peer company comparison to enhance the reference value of results, optimising visualisation styles to improve readability, and expanding more advanced financial indicator calculation to enrich the analysis depth.
My main personal contributions include determining the analysis subject of Microsoft stock after considering the representativeness of the company and the completeness of data, designing the overall project workflow in combination with the course requirements, organising all project files in a standardized manner to ensure the clarity of the project structure, building the GitHub repository according to the methods taught in ACC102 tutorials, completing the README document to explain the project details clearly, and finishing this reflection report independently to summarize the whole project process. Through this task, I mastered the whole process of financial data acquisition and analysis, understood the standard academic project submission norms on GitHub, and significantly improved my practical Python programming ability and problem-solving ability in data analysis.
